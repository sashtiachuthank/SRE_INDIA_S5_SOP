DB2 Fix pack Installation

Goal¶

This article helps you to pursue the DB2 fixpack installation in appropriate hosts.

Procedure¶

üìå No downtime is required for this activity

Pre-installation task:¶

  1. We should have the root access to start DB2 fix pack activity. sudo su -
  2. Check the file system space available. a) The Installation Path: - /opt/IBM/db2 (At least 2GB free space should be available). b) /tmp FS try to keep 4GB free space to download the software. c) Please check the other db2 Filesystems. Better to keep all the db2 Filesystems below 80% used. df -h ---> for Linux df -gt ---> for AIX

Pre fixpacki nstall steps: 1.Manual backup of Database Configuration parameters: Create directory of DB2 fix pack. a) mkdir fixpack_V11.5.8 b) cd fixpack_V11.5.8 c) db2licm -l > db2licm_before.out d) db2level > db2level_before.out e) db2 get dbm cfg > dbmcfg_before.out f) db2 list db directory > dbdir_before.out g) db2 list node directory > nodedir_before.out h) db2set -all > db2set_before.out i) db2 connect to ; db2 get db cfg for dbname > dbcfg__before.out j) db2 list tablespaces show detail > tablespaces_before.out k) db2look -d dbname -e -x -o db2look_before.out l) db2support . -d -c -s m) crontab -l > crontab_before.out 2. Perform database dackup. a) Application team need to stop all the connected applications. Once they stopped, we will get the output like below. db2 list active databases (or) db2 list applications

b) Deactivate the database

db2 deactivate db

c) Take offline database backup to TSM
db2 backup db use TSM Steps for Installation: 1.Stop db2 services. Switch to each instance and execute the following commands. a) Application team bring down the applications which are connected to the database. Make sure we don‚Äôt have any connections by using below command. db2 list active databases (or) db2 list applications b) Deactivate the database using ‚Äúdb2 deactivate db ‚Äù c) Stop the instance by using db2stop d) ipclean -a e) ps -ef|grep -v grep |grep -i db2sysc -----> (Check if db2 processes still existing or not)

2.As root, we have to install the required fixpack a) Go to db2 software directory where we copied & extract it. tar -zxvf

b) Go to the ‚Äúuniversal‚Äù path and do prereqcheck whether software is compatible with OS or not. cd universal ls -ltr ./db2prereqcheck -v 11.5.8.0

c) Once all requirements matched, we can install the db2 fixpack db2level ------> To know the db2 version db2ls -------> List out db2 softwares

d) Make a new directory under this path /opt/IBM/db2 cd /opt/IBM/db2 mkdir V11.5.8.0 chmod 755 V11.5.8.0 e) Now go to the extracted software path: cd universal ./installFixPack -b (old path we will get from db2ls) Input while running above command as below screen. 1.Yes 2.Yes 3./opt/IBM/db2/V11.5.8.0

f) Validate whether db2 software is properly installed or not . cd /opt/IBM/db2/V11.5.8.0/bin ./db2val

g) Check db2level

Post Installation Steps: 1. Update each db2 instances as root cd /opt/IBM/db2/V11.5.8.0/instance ./db2iupdt -u -----> (To know fenced user by using db2pd -fmp)

  1. Switch to instance and update the database. cd /opt/IBM/db2/V11.5.8.0/bin ./db2updv115 -d

  1. Perform the bind files after updating databases. ‚Ä¢ cd /home/sqllib/bnd ‚Ä¢ db2 connect to ‚Ä¢ db2 BIND @db2ubind.lst BLOCKING ALL GRANT PUBLIC ‚Ä¢ db2 BIND @db2cli.lst BLOCKING ALL GRANT PUBLIC ‚Ä¢ db2 BIND @db2schema.bnd BLOCKING ALL GRANT PUBLIC ‚Ä¢ db2 BIND @db2exfmt.bnd BLOCKING ALL GRANT PUBLIC ‚Ä¢ db2rbind -l db2rbind_dbname.log all
  2. Make sure all table spaces state should be normal. db2 connect to db2 list tablespaces show detail |grep -i state

5.Take all parameter backups after fixpack activity completed.

‚Ä¢ db2licm -l > db2licm_after.out ‚Ä¢ db2level > db2level_after.out ‚Ä¢ db2 get dbm cfg > dbmcfg_after.out ‚Ä¢ db2 list db directory > dbdir_after.out ‚Ä¢ db2 list node directory > nodedir_after.out ‚Ä¢ db2set -all > db2set_after.out ‚Ä¢ db2 connect to ; db2 get db cfg for dbname > dbcfg__after.out ‚Ä¢ db2 list tablespaces show detail > tablespaces_after.out ‚Ä¢ db2look -d dbname -e -x -o db2look_after.out ‚Ä¢ db2support . -d -c -s ‚Ä¢ crontab -l > crontab_after.out

NOTE: Need to compare parameters backups before and after and they should be expected values. Ex: diff db2licm_before.out db2licm_after.out 6. Once everything is good from database side, we need to ask application team to start the applications and validate from their end. If everything is fine form application side, we are good to signoff this task.
